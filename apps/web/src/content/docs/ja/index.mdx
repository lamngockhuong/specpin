## Specpinができること

Specpinはビジネス仕様（ルール、説明、受け入れ基準）を*動作中*のウェブUIの要素に直接貼り付け、
ホバーまたは閲覧中にブラウザ内でレンダリングします。Specpinはコードジェネレーターでは**ありません**：
すでに持っているインターフェースに、Gitバージョン管理された生きたドキュメントを固定するものです。

<CardGrid>
  <Card title="ライブ要素にspecを固定" icon="pin">
    レジリエントなフィンガープリントマッチング（test-id、aria、selector、xpath、テキスト、
    位置）により、リファクタリング後もspecが要素に固定されたままになります。
  </Card>
  <Card title="3つの表示モード" icon="laptop">
    ツールチップ、ページ内サイドバー、またはドラッグ可能なモーダルとしてspecを読めます。
    spec単位またはページ全体で切り替えられます。
  </Card>
  <Card title="その場でキャプチャ" icon="add-document">
    ページを離れずに要素をクリックしてspecを作成できます。Markdownの説明、
    ビジネスルール、タグ、多言語タブをサポートします。
  </Card>
  <Card title="書き込み可能なローカルプロジェクト" icon="document">
    sidecarを起動しなくてもspecの編集、キャプチャ、作成、グループzipエクスポートができます。
  </Card>
  <Card title="マルチプロジェクト、origin別" icon="open-book">
    1つのextensionで複数のプロジェクトを同時に扱い、ドメインで各ページに振り分けます。
    プロジェクトごとに個別に有効・無効を切り替えられます。
  </Card>
  <Card title="デフォルトでセキュア" icon="approve-check">
    sidecarはデフォルトでlocalhostにバインドし、すべてのリクエストをbearerトークンで認証し、
    extensionのoriginのみを受け付けます。チームに必要なときはHTTPS経由のリモートsidecarを
    オプトインできます — それはあなたが運用するサーバーのままです。
  </Card>
</CardGrid>

## 動作の仕組み

<div class="specpin-flow" not-content>
  <div class="step">
    <span class="n">1</span>
    <h3>specはリポジトリに存在する</h3>
    <p>specはリポジトリの<code>.specs/</code>ディレクトリにあるJSON：バージョン管理され、
    PRでレビューでき、差分確認も可能です。</p>
  </div>
  <div class="step">
    <span class="n">2</span>
    <h3>sidecarが配信する</h3>
    <p><code>specpin serve</code>はトークン認証付きのlocalhost APIで<code>.specs/</code>を
    ライブリロード付きで公開します。</p>
  </div>
  <div class="step">
    <span class="n">3</span>
    <h3>extensionがレンダリングする</h3>
    <p>ブラウザextensionが各specをライブ要素にマッチさせ、ページ上にレンダリングします。</p>
  </div>
</div>

```text
.specs/ (リポジトリ内)  -->  specpin serve (localhost HTTP + SSE)  -->  ブラウザextension (マッチ + レンダリング)
```

## クイックスタート

```bash
# 1. ワークスペースの依存関係をインストールしてすべてをビルド
pnpm install
pnpm build

# 2. GoのsidecarをビルドInApps/cliから
cd apps/cli && make build      # -> bin/specpin

# 3. プロジェクトリポジトリで：specのスキャフォールドと配信
specpin init                   # .specs/manifest.jsonを作成
specpin serve                  # localhost URL + bearerトークンを表示

# 4. extensionをビルド（unpacked）して接続
#    Chrome:  pnpm --filter @specpin/extension build         -> .output/chrome-mv3
#    Firefox: pnpm --filter @specpin/extension build:firefox -> .output/firefox-mv2
```

表示されたURLとトークンをextensionに貼り付け、アプリを開くと各要素にspecがレンダリングされます。
