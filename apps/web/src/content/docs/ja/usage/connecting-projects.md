Specpinはプロジェクトに2つの方法で接続します：**sidecar**接続（実行中の`specpin serve`インスタンス）または**ローカルプロジェクト**（ブラウザに保存されたspec）。

## 新しいプロジェクトを追加する

Specpin popupまたはサイドパネルを開き、設定の歯車（右上）または**+ 新しいプロジェクト**ボタンをクリックします。2つの選択肢が表示されます：

### sidecar接続

実行中の`specpin serve`インスタンスに接続します。

1. プロジェクトディレクトリで`specpin serve`を実行します（詳細は[CLI](/ja/sidecar/cli/)を参照）。URLとトークンが表示されます：
   ```
   Specpin sidecar running.
     URL:     http://127.0.0.1:51234
     Token:   2da0480c...
   ```
2. extensionで**Sidecar**を選択します。
3. **sidecar URL**を貼り付けます（例：`http://127.0.0.1:51234`）。
4. **ラベル**を追加します（任意、例：「Acme CRM」）。
5. **トークン**を貼り付けます。
6. **テストしてプロジェクトを追加**をクリックします。接続に成功すると、プロジェクトがOptionsページのリストに表示されます。

URLはlocalhostアドレス（`http://127.0.0.1:<port>`または`http://localhost:<port>`）でも、HTTPS経由の**リモート**sidecar（例：`https://specs.example.com`）でも構いません。リモートsidecarは`https://`が必須です — リモートホストへのプレーンな`http://`は拒否されます。IPのみでドメインのないホストでも利用できます。そのIPをSANに持つ証明書（内部CA）を使った`https://<ip>`か、`http://localhost:<port>`へのSSHトンネルを使ってください。リモート接続を追加すると、ブラウザがそのホストへのアクセス許可を求めます。接続するには許可してください。接続を削除すると、その許可は取り消されます。リバースプロキシの背後でsidecarを実行する方法は[リモートマシンで配信する](/ja/sidecar/cli/)を参照してください。

:::tip
各sidecarは独自のポートで動作します。複数のプロジェクトを同時に配信するには、プロジェクトAで`specpin serve --port 51001`、プロジェクトBで`specpin serve --port 51002`を実行し、それぞれ独自のトークンで両方の接続を追加します。
:::

:::note
あなたが同じプロジェクトを編集している間にチームメンバーがspecを変更した場合、保存は「他の場所で変更された」という通知とともに拒否され、プロジェクトが再読み込みされます — 確認してから再度保存してください。これにより、ある書き込みが別の書き込みを黙って上書きすることを防ぎます。
:::

### ローカルプロジェクト

sidecarなしでプロジェクトを作成します。specは`browser.storage.local`に保存され、`.specs.zip`としてエクスポートできます。

1. **ローカルプロジェクト**を選択します。
2. **プロジェクト名**を入力します。
3. （任意）**ドメイン**を入力します（カンマ区切り、例：`localhost:3000, example.com`）。
4. ドメインを空白のままにして**すべてのサイトに適用**もチェックしない場合、プロジェクトはどのページも配信しません（後でドメインを追加できます）。
5. **作成**をクリックします。

ローカルプロジェクトは、ページがドメインに一致する場合にpopupとサイドパネルに表示されます。sidecarプロジェクトと同じ方法でspecをキャプチャ・編集できます。

## マルチプロジェクトルーティング

1つのextensionで複数のプロジェクトを配信できます。specはプロジェクトのmanifestの`domains`にそのページのoriginが含まれている場合にのみ表示されます。ドメインが設定されていないプロジェクトはデフォルトで非アクティブです（そうでなければspecがどこにでも表示されてしまいます）。接続設定で**すべてのサイトに適用**をチェックして有効にします。

## プロジェクトごとの有効/無効

Optionsページの各接続には**有効**/**無効**トグルがあります。プロジェクトを無効にすると、どこでもspecが非表示になりSSEウォッチが停止しますが、後で再有効化できるよう接続はリストに残ります。

Specpinがページ（グローバルトグル）でオフになっている場合、すべてのプロジェクトが無視されます。プロジェクトごとのトグルはグローバルのオン/オフとは独立して動作します。

## 接続の管理

extension Optionsページを開きます（popupまたはサイドパネルの設定歯車をクリック）。

- **編集**：URL、ラベル、またはトークンを変更します。トークンフィールドを空白のままにすると現在のものが保持されます。編集後に**変更を保存**をクリックします。
- **再接続**：sidecarが再起動されたかトークンが変更された場合に接続を再テストします。
- **削除**：接続を削除します。確認ダイアログが最初に表示されます。

ローカルプロジェクトには**名前変更**（プロジェクト名とドメインの変更）と**エクスポート**（プロジェクトのspecの`.specs.zip`のダウンロード）アクションが表示されます。sidecarプロジェクトにも**エクスポート**（ライブキャッシュからバンドルを作成）が表示されます。

ローカルプロジェクトの場合、Optionsページには**ローカル**のプロベナンスラベルと固定されたサイト（ドメインが固定されていない場合は「すべてのサイト」）が表示されます。
