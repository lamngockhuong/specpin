## 外観

これらの設定はSpecpin自身のインターフェースの外観を制御します。すべてのSpecpinサーフェス（popup、サイドパネル、options、およびページ内のツールチップ/サイドバー/モーダル）に適用されます。

### テーマ

Specpinの表示方法を選択します：

- **システムデフォルト**：OSのライト/ダーク設定に従います。
- **ライト**：常にライトテーマを使用します。
- **ダーク**：常にダークテーマを使用します。

テーマは開いているすべてのSpecpinサーフェスに即座に反映されます。

### 言語

SpecpinのUI言語を選択します：

- **システムデフォルト**：ブラウザのUI言語を使用します（利用できない場合は英語にフォールバック）。
- **English**：英語UIを強制します。
- **Tiếng Việt**：ベトナム語UIを強制します。

この設定はspecコンテンツの言語（popupとサイドパネルの**言語**ドロップダウン）とは独立しています。UI言語を変更してもspecの表示言語は変わりません。

## ツールバーアイコン（Chromeのみ）

Chromeでは、Specpinツールバーアイコンをクリックしたときの動作を選択できます：

- **popupを開く**：デフォルトの動作。
- **サイドパネルを開く**：アイコンをクリックしてサイドパネルを直接開きます。

:::note
Firefoxでは、ツールバーアイコンは常にpopupを開きます。サイドパネルは**表示 -> サイドバー -> Specpin**から開いてください。
:::

## サポートとフィードバック

OptionsページにはプロジェクトのGitHubへのリンクがあります：

- **不具合を報告**：GitHub Issuesページを開きます。
- **質問する**：GitHub Discussionsページを開きます。

これらを使って不具合の報告、機能リクエスト、またはヘルプの要請ができます。

## キーボードショートカット

これらのショートカットはSpecpinが読み込まれている任意のページで動作します：

| ショートカット | 操作 |
|----------|--------|
| `Alt+Shift+S` | Specpinのオン/オフ |
| `Alt+Shift+M` | 表示モードの切り替え（ツールチップ -> サイドバー -> モーダル） |
| `Alt+Shift+C` | キャプチャモードに入る（`Esc`でキャンセル） |
| `Alt+Shift+N` | 一致したspec間を巡回する（各要素を点滅、最後で先頭に戻る） |
| `Alt+Shift+G` | デフォルトのガイドツアーを開始 / 停止 |

ショートカットは常に有効です。Optionsページからカスタマイズすることはできません（変更が必要な場合はブラウザのextensionショートカット設定を使用してください）。

## プロジェクトごとの設定

Optionsページの各接続済みプロジェクトには、以下のコントロール付きの行があります：

- **有効/無効トグル**：削除せずにプロジェクトをオン/オフします。
- **編集**：URL、ラベル、またはトークン（sidecar）、もしくは名前とドメイン（ローカル）を変更します。
- **再接続**：sidecar接続を再テストします（sidecarのみ）。
- **エクスポート**：プロジェクトのspecの`.specs.zip`をダウンロードします（ローカルとsidecar）。
- **名前変更**：プロジェクト名とドメインを変更します（ローカルのみ）。
- **削除**：接続を削除します。

詳細は[プロジェクトの接続](/ja/usage/connecting-projects/)を参照してください。

## 手動spec（sidecarなし）

OptionsページにはSpecpinを実行せずにspecをインポートできる**手動spec（sidecarなし）**セクションがあります。ファイルからspecを読み込む（`manifest.json` + 1つ以上の`*.spec.json`を選択）か、バンドルJSONを貼り付けます。

各インポートはバッチを追加します。読み込まれたバッチはボタンの下に表示され、インポートごとに1枚のカードに**エクスポート**、**名前変更**、**削除**アクションが付きます。**すべての手動specを消去**をクリックしてリストを空にします。

バンドルJSONの形式と`specpin bundle`コマンドについては[実行ガイド](/ja/guide/getting-started/)を参照してください。
