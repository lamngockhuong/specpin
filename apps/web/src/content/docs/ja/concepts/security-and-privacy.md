Specpinはローカルファーストで設計されています。デフォルトでは、specはマシンの外に出ることなく、sidecarサーバーはブラウザextensionからの接続のみを受け付けるように強化されています。チームは任意で、同じsidecarをHTTPSリバースプロキシの背後にある自分のホストで実行できます。その場合でも、specはあなたが運用するサーバーにのみ送られ、Specpinのサービスに送られることはありません。

## ローカルファーストアーキテクチャ

すべてのデータはローカルマシンを経由します：

1. **specはリポジトリに存在します**。`.specs/*.json`ファイルとして。他のソースコードと同様にGitでバージョン管理されます。
2. **デフォルトでは、sidecar（`specpin serve`）は`127.0.0.1`にバインドします。** localhostで、自動選択されたポート（または`--port`で指定したポート）でリッスンします。リモートインターフェースを明示的にバインドしない限り、外部トラフィックは到達できません。
3. **ブラウザextensionはlocalhost HTTP経由でspecを取得します。** bearerトークンを使って`http://127.0.0.1:<port>`に接続します（リモートsidecarの場合は自分の`https://`リバースプロキシに接続します）。
4. **書き込みはディスク上の`.specs/`に戻ります。** specをキャプチャまたは編集すると、extensionがsidecarに送り、sidecarがアトミックにpretty-printされたJSONファイルを書き込みます。変更はすぐに`git diff`に表示されます。

Specpinのどのコンポーネントも、Specpinが運用するサーバーにデータを送信しません。クラウドサービス、テレメトリ、アナリティクスはありません。specデータは、あなたが実行するsidecar — デフォルトではlocalhost、オプトインすれば自分のリモートホスト — にのみ送られます。

## sidecarのセキュリティモデル

sidecarは不正アクセスを防ぐために強化されています：

### localhostバインディング（デフォルト）

デフォルトでは、sidecarは`127.0.0.1`にバインドします。マシン上のプロセスのみがアクセスできます。`--port`で上書きしない限り、利用可能な高ポートから自動選択されます。

チーム利用では、`--host <addr>`が非loopbackインターフェースにバインドします。これは生の**プレーンテキストかつトークンのみ**のポートをネットワークに直接公開し（リバースプロキシが自動的に経路へ入るわけではありません）、sidecarは警告を表示します。そのポートはfirewallで保護し、前段にHTTPSリバースプロキシを置く必要があります。リモートクライアントは常にそのプロキシ経由のHTTPSで接続します — extensionはmixed contentとしてプレーンテキストのリモート接続をブロックします。

### bearerトークン認証

すべてのリクエスト（ヘルスチェックを除く）に`Authorization: Bearer <token>`ヘッダーが必要です。トークンは`specpin serve`実行時に表示されます：

```
Specpin sidecar running.
  URL:     http://127.0.0.1:51234
  Token:   2da0480c...
```

トークンをコピーしてextensionの接続設定に貼り付けます。extensionはバックグラウンドサービスワーカーに安全に保存し、すべてのリクエストと一緒に送信します。正しいトークンがないと、すべてのリクエストが`401 Unauthorized`で拒否されます。

:::caution
トークンはターミナルに表示されます。マシンへの物理的またはリモートアクセス権を持つ人がターミナル出力を見ると、sidecarが動作中に接続できます。トークンをパスワードと同様に扱ってください — これはブラウザ以外のネットワーククライアントに対する**唯一**の認可境界です（CORSはブラウザのみを制約します）。デフォルトではトークンは`specpin serve`を再起動するたびに変わります。再起動で全員の認証が無効にならないよう、`--token` / `SPECPIN_TOKEN`で安定したトークンを固定できます（リモート/チーム用sidecar向け）。ただしその場合は、配布とローテーションを慎重に行ってください。
:::

### CORS：extensionのoriginのみ

sidecarはブラウザextensionのoriginからのリクエストのみを受け付けます：

- `chrome-extension://*`
- `moz-extension://*`
- `safari-web-extension://*`

ウェブページ（`localhost`上のものも含む）は拒否されます。これにより、攻撃者がトークンを知ったとしても、悪意のあるウェブサイトがspecを盗んだり偽のspecをリポジトリに書き込んだりするのを防ぎます。

### パストラバーサル防止

すべてのファイル書き込みは`.specs/`ディレクトリに限定されます。sidecarはパストラバーサル攻撃（例：`../../etc/passwd`）を防ぐためにすべてのファイルパスを検証します。`.specs/`の外に書き込もうとするリクエストは拒否されます。

### アトミックなpretty-print書き込み

spec書き込みはアトミックです：sidecarは一時ファイルに書き込み、JSONを検証してから名前を変えて所定の場所に置きます。書き込みの途中でプロセスがクラッシュしても、`.specs/`ディレクトリが壊れた状態になることはありません。すべてのJSONはpretty-printされるので`git diff`が読みやすいままです。

## extensionのセキュリティ

ブラウザextensionはManifest V3 extension（Chrome）とManifest V2（Firefox）として、セキュリティのベストプラクティスに従って構築されています：

- **トークンはバックグラウンドサービスワーカーに留まります。** extensionのOptionsページはDOMにトークンをエコーしません。接続ステータスのクエリはトークンをリークできません。
- **コンテンツスクリプトはサンドボックス化されています。** extensionのコンテンツスクリプト（ウェブページ上で動作）は接続を変更したりトークンを読み取ったりできません。接続設定を変更できるのはpopup、サイドパネル、またはoptionsページからの特権メッセージのみです。
- **書き込みはoriginでゲートされています。** specをキャプチャする際、extensionは対象プロジェクトの`domains`が現在のページのoriginをカバーしていることを確認します。`localhost:3000`を配信するsidecar接続は`example.com`からのキャプチャを受け付けません。

## ローカルプロジェクト（手動spec）

extensionはsidecarの代わりにブラウザのextensionストレージ（`browser.storage.local`）にspecを保存する**ローカルプロジェクト**をサポートします。ローカルプロジェクトはoriginでバインドされています：`localhost:3000`のspecは`example.com`のspecから隔離されています。ローカルspecは`.specs.zip`バンドルとしてエクスポートしてリポジトリにコミットするか、別のマシンに再インポートできます。

ローカルプロジェクトはブラウザプロファイルにプライベートであり、`.specs.zip`ファイルを明示的にエクスポートして共有しない限りクラウドに同期されません。

## マルチプロジェクト信頼モデル

extensionを複数のsidecarインスタンスに接続できます（例：プロジェクトごとに1つ）。各接続には独自のURLとトークンがあります。トークンは別々に保存され、プロジェクト間で共有されることはありません。あるプロジェクトのトークンが漏洩しても、他のプロジェクトには影響しません。

## データの保存場所

| データ | 場所 | 共有? |
|------|----------|---------|
| spec（sidecarプロジェクト） | リポジトリ内の`.specs/*.json` | Git経由（あなたが管理） |
| spec（ローカルプロジェクト） | `browser.storage.local` | なし（ブラウザプロファイルのみ） |
| 接続URLとトークン | extensionバックグラウンド（メモリ内 + `browser.storage.local`） | なし |
| UI設定（テーマ、言語） | `browser.storage.local` | なし |
| 個人の表示オーバーライド | `browser.storage.sync` | サインイン済みブラウザ間（ブラウザ同期経由） |

## よくある質問

**データはどこかに送られますか？**  
私たちには送られません。デフォルトでは、すべてのspecデータはローカルマシンのみを経由します — sidecarは`127.0.0.1`にバインドし、extensionはlocalhost経由で接続します。リモートsidecarをオプトインした場合、specはそのホストにのみ送られ、それは**あなた**が運用するものです。Specpinのサーバーは存在せず、テレメトリもありません。

**ウェブサイトがspecを読み取ることはできますか？**  
いいえ。sidecarはウェブoriginからのリクエストをCORS経由で拒否します。ブラウザextensionのみがspecを取得できます。

**トークンが表示されたターミナルを誰かに見られた場合はどうなりますか？**  
その人はsidecarが動作中に接続して、そのプロジェクトのspecを読み書きできます。トークンは`specpin serve`を再起動するたびに変わります。スクリーン共有中やロックされていないマシンでターミナルを表示したままにしないでください。

**ローカルプロジェクトはクラウドに同期されますか？**  
いいえ。ローカルプロジェクトはブラウザプロファイルにプライベートな`browser.storage.local`に保存されます。`.specs.zip`としてエクスポートして手動でファイルを共有しない限り同期されません。

**リモートサーバーでSpecpinを使用できますか？**  
はい。リモートホストでsidecarを実行し（同じホストにHTTPSリバースプロキシを置いてloopbackのままにするか、`--host` + firewallを使用）、安定した`--token`を固定し、extensionをプロキシの`https://` URLに接続します。extensionは接続を追加する際にその1つのoriginへのアクセスを要求し、削除する際に取り消します。リモートはHTTPSが必須です（プレーンテキストのリモートはmixed contentとしてブロックされます）。Caddy/nginxの例と脅威モデルについては、実行ガイドの「Serve on a remote machine」セクションを参照してください。
