## Specpin làm gì

Specpin gắn các đặc tả nghiệp vụ (quy tắc, mô tả, tiêu chí nghiệm thu) trực tiếp
lên các phần tử của một giao diện web *đang chạy*, rồi hiển thị ngay trong trình
duyệt khi bạn rê chuột hoặc duyệt trang. Specpin **không** sinh code: nó ghim tài
liệu sống động, được quản lý phiên bản bằng Git, lên chính giao diện bạn đang có.

## Cách hoạt động

<div class="specpin-flow" not-content>
  <div class="step">
    <span class="n">1</span>
    <h3>Đặc tả nằm trong repo</h3>
    <p>Đặc tả là JSON trong thư mục <code>.specs/</code> của repo: quản lý phiên
    bản, duyệt qua PR, và so sánh diff được.</p>
  </div>
  <div class="step">
    <span class="n">2</span>
    <h3>Sidecar phục vụ chúng</h3>
    <p><code>specpin serve</code> mở <code>.specs/</code> qua một API localhost
    có xác thực token kèm tải lại trực tiếp.</p>
  </div>
  <div class="step">
    <span class="n">3</span>
    <h3>Extension hiển thị chúng</h3>
    <p>Extension trình duyệt khớp mỗi đặc tả với một phần tử sống và hiển thị nó
    ngay trên trang.</p>
  </div>
</div>

```text
.specs/ (trong repo)  -->  specpin serve (localhost HTTP + SSE)  -->  extension trình duyệt (khớp + hiển thị)
```

## Bắt đầu nhanh

```bash
# 1. Cài deps workspace và build mọi thứ
pnpm install
pnpm build

# 2. Build sidecar Go (trong apps/cli)
cd apps/cli && make build      # -> bin/specpin

# 3. Trong repo dự án của bạn: khởi tạo và phục vụ đặc tả
specpin init                   # tạo .specs/manifest.json
specpin serve                  # in ra URL localhost + bearer token

# 4. Build extension (unpacked) và kết nối
#    Chrome:  pnpm --filter @specpin/extension build         -> .output/chrome-mv3
#    Firefox: pnpm --filter @specpin/extension build:firefox -> .output/firefox-mv2
```

Dán URL và token vừa in vào extension, mở ứng dụng của bạn, và đặc tả sẽ hiển
thị trên đúng phần tử của nó.
